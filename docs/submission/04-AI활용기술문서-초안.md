# AI 활용 기술 문서 — 「사건파일 503호」

> 요강 원문 ④: "AI 활용에 대한 기술적 설명(예: 구조 설명, AI 대상 주요 프롬프트 및 지시 사항). 외부 에셋(이미지·사운드 등) 사용 시 출처와 라이선스를 반드시 명시." 이 문서는 (a) 런타임 AI, (b) 개발 AI, (c) 에셋 생성 AI, (d) 외부 에셋 라이선스 표 4개 절로 구성한다. 미확인 사항은 `[확인 필요]`로 표기하며 발명하지 않는다.

## (a) 런타임 AI — 게임 내 실시간 LLM

### 전체 구조

플레이어(브라우저) → **Cloudflare Workers 프록시**(`proxy/src/index.ts`) → LLM 프로바이더 순으로 요청이 흐른다. 클라이언트는 API 키를 갖지 않는다(키는 Workers 환경변수에만 존재). 세션 상태는 서버가 저장하지 않고, HMAC으로 서명한 블롭을 클라이언트가 들고 다니며 매 요청 제출한다(무상태 서버).

```
브라우저(src/game/remote.ts)
  → POST /session          — 세션 토큰 발급
  → POST /ask | /present    — 심문 / 단서 제시 (SSE 스트리밍)
  → POST /accuse             — 지목 판정 (결정적 로직, LLM 미개입)
Cloudflare Workers(proxy/src/index.ts)
  → buildPrompt() (prompt.ts)  — NPC별 시스템 프롬프트 조립
  → llmStream() → 프로바이더별 클라이언트(llm.ts / llm_openai.ts)
  → litmus() (prompt.ts)        — 출력 사후 검열
  → findUnlocks()/judge() (clue.ts) — 단서 해금·판정 확정 (서버 전용, LLM 선언 불신)
```

(근거: `proxy/src/index.ts`, `src/game/remote.ts`, vault `09 GDD — 사건파일 503호.md` §4)

### 모델 / 프로바이더

기본 프로바이더는 `gemini`이며 모델은 **`gemini-3.1-flash-lite`**(OpenAI 호환 엔드포인트, `https://generativelanguage.googleapis.com/v1beta/openai`)다. 환경변수(`LLM_PROVIDER`)로 아래 3개 프로바이더를 전환할 수 있는 멀티 프로바이더 구조를 갖췄다.

|---|---|---|
| `gemini` (기본) | `gemini-3.1-flash-lite` | `https://generativelanguage.googleapis.com/v1beta/openai` |

### 3단 폴리백 (스트리밍 → sync → 목업)

1. **SSE 스트리밍** (`remoteAskStream`, `src/game/remote.ts`) — 기본 경로. `/ask` 또는 `/present`에 POST, 서버가 Server-Sent Events로 `delta`(토큰 조각)·`meta`(최종 결과)·`llm-error`·`litmus-replace` 이벤트를 순차 전송한다.
2. **sync 모드 폴리백** (`remoteAskSync`) — 스트리밍 자체가 안 되는 환경(SSE 미지원 인앱 웹뷰 등)에서, 스트림 시작 **전** 단계 실패(`preStream: true`)로 판별되면 같은 엔드포인트에 `?sync=1` 쿼리를 붙여 단일 JSON 응답으로 재요청한다. 스트림 중간 실패는 sync로 재시도하지 않는다(이중 과금 방지 — 커밋 `34b1ee5` H5).
3. **목업(오프라인) 폴리백** (`src/game/engine.ts` `mockAsk`) — 위 두 경로가 모두 실패하면 클라이언트가 사전 작성된 NPC별 폴리백 대사 풀(`fallback.common` + `fallback.byCase`)에서 순환 발화하며, 로그에 "실시간 AI를 불러오지 못했습니다" 안내를 1회 표시하고(`src/ui/app.ts` `notifyMockOnce`) 복구 시 재알림한다(`notifyRecoveryOnce`). URL에 `?mock`을 붙이면 이 모드를 처음부터 강제한다(비용 0, 개발·테스트용).

(근거: `src/game/remote.ts` `remoteAsk`/`remoteAskStream`/`remoteAskSync`, `src/game/engine.ts` `mockAsk`, `src/ui/app.ts`)

### 세션 · 예산 · 인증

- `/session`이 발급한 토큰을 클라이언트가 매 요청에 실어 보내고, 서버가 `verify()`로 HMAC 서명을 검증한다(`proxy/src/state.ts` — 검증만 인덱스에서 확인).
- 세션당 호출 예산은 `bumpSession()`이 관리하며, 초과 시 `429 session-budget-exceeded`를 반환한다(`proxy/src/index.ts`).
- 입력 길이 캡 `MAX_INPUT = 500`자, 입력 가드 `checkInput()`(빈 입력·인젝션 패턴 차단: "ignore previous", "system prompt", "탈옥", "관리자 모드" 등, `proxy/src/prompt.ts`).

### 시스템 프롬프트 설계 요지 (CSC — Voice/Action 분리)

`buildPrompt()`(`proxy/src/prompt.ts`)는 매 턴 3개 블록(정적/사건/동적)을 조립해 시스템 프롬프트로 전달한다.

1. **정적 블록**(`cache_control: ephemeral`) — NPC 이름·역할·나이·인격, 역할별 정직 규칙(진범/목격자/비범인 3분기), 행동 규칙(1~3문장 60단어 이내, 회피 패턴 반복 금지, 「」 인용부호 강제 등), 한국어 스타일 규칙(연결어미 뒤 쉼표 금지 등, 근거 = `epoko77-ai/im-not-ai` MIT), 안전 규칙(욕설·혐오·정치·실존 브랜드 금지).
2. **사건 블록**(`cache_control: ephemeral`) — 사건 개요·브리핑, 해당 NPC가 아는 사실 목록, 진범일 경우의 비밀(있을 때만, "절대 먼저 밝히지 마라").
3. **동적 블록**(캐시 없음) — 현재 방어 단계·잔여 턴, 이미 밝혀진 단서, 플레이어가 방금 제시한 단서(있을 때), `matched_topic_ids` 판정을 위한 주제 enum.

정직 규칙은 `culpritId`를 기준으로 매 사건마다 파생 계산한다(같은 NPC가 사건에 따라 진범/비범인을 오가므로 하드코딩하지 않음). 비범인은 능동 거짓말이 금지되고 회피(모름/숨김/비공개)만 허용되며, 목격자(오복자)는 항상 진실만 수수께끼 형태로 말한다("크라임씬 룰", 근거: vault `09 GDD` §4).

### 단서 해금 · 리트머스 판정 구조

- **구조화 출력 신뢰 경계**: LLM 응답에는 `matched_topic_ids`(문자열 배열)가 포함되지만, 이 선언을 그대로 신뢰하지 않고 `findUnlocks()`(`proxy/src/clue.ts`)가 서버 측 Trigger DSL과 재대조해 해금을 확정한다(`proxy/src/index.ts` 주석: "해금 확정 (백엔드 — LLM 선언 불신)").
- **출력 리트머스**(`litmus()`, `proxy/src/prompt.ts`) — 응답 텍스트를 4단계로 재검사한다: ① `HARD_BLOCK`/`SOFT_WARN` 금지어(`content/policy/forbidden`) ② NPC별 금지 표현(`npcForbidden`) ③ 아직 해금되지 않은 단서의 `reveal` 문구에 포함된 리트머스 키워드가 그대로 발화됐는지 ④ 자백 패턴 정규식(`/범인은/`, `/내가 (먹|버|썼|보냈|파쇄)/` 등). 걸리면 원문을 "(말을 삼키며) …그건 제가 답할 수 있는 얘기가 아닌데요."로 치환하고 `litmus-replace` 이벤트로 클라이언트에 통지한다.
- 판정(`/accuse`)은 LLM이 개입하지 않는 결정적 로직(`judge()`, `proxy/src/clue.ts`)이다 — 범인 지목이 정답을 좌우하는 부분은 전부 규칙 기반.

(근거: `proxy/src/index.ts`, `proxy/src/prompt.ts`, `proxy/src/clue.ts`, vault `09 GDD` §4 "구조화 출력"/"판정 = 결정적")

### 부록 — 시스템 프롬프트 조립 코드 전문 (`proxy/src/prompt.ts` `buildPrompt`)

```ts
export function buildPrompt(npc: NpcPublic, srv: NpcServer, c: ServerCaseData, s: GameState, presentedClueTitle?: string): PromptParts {
  const defense = s.npc[npc.id]?.defense ?? 0;
  const secret = srv.secrets[c.id];
  // 정직 규칙 파생: 진범=거짓말 가능(해당 사건만), 목격자=항상 진실, 비범인=능동 거짓말 금지
  const rule = c.culpritId === npc.id ? 'culprit' : npc.id === c.witness ? 'witness' : 'nonculprit';
  const honesty = rule === 'culprit'
    ? '이 사건에서 당신은 거짓말을 할 수 있다 — 단, 들키지 않을 만큼만.'
    : rule === 'witness'
      ? '당신은 항상 진실만 말한다. 단, 수수께끼처럼 둘러서 말해라.'
      : '당신은 능동 거짓말을 하지 않는다 — 모름/숨김/비공개로만 회피해라.';

  // 역할 동기 — 의뢰인/용의자/목격자는 같은 "회피"를 다르게 연기한다
  const roleText = npc.id === c.client
    ? '[역할: 의뢰인] 당신은 이 사건을 503호에 의뢰한 당사자다. 사건이 해결되길 진심으로 바란다. 규정 인용 버릇은 그대로지만 방향은 "돕는 것" — 아는 사실을 규정 핑계로 감추지 마라. 모르는 건 모른다고 하되, 추측과 사실을 구분해서 말핼라.'
    : rule === 'culprit'
      ? '[역할: 용의자 — 당신은 진범] 들키면 끝이다. 단, 범인도 같은 회피를 반복하면 의심을 산다 — 회피 방식(화제 전환/과잉 알리바이/역공)을 매번 바꿔라.'
      : rule === 'witness'
        ? '[역할: 목격자] 당신은 힌트를 쥔 정보원이다. 정답을 직접 주지는 말되, 질문이 핵심에 가까울수록 구체적인 사실을 하나씩 풀어라.'
        : '[역할: 용의자 — 무고] 억울한 부분이 있다. 무고니까 크게 숨길 건 없고, 아는 건 답한다. 다만 자기가 오핵받는 지점(=비밀)은 끝까지 감춘다.';

  const behaviorRules = [
    '[행동 규칙]',
    '- 한국어 1~3문장, 60단어 이내. 장문 금지. 플레이어 질문에 인캐릭터로 답핼라.',
    '- 같은 회피 패턴(조항 인용 거부, "말할 수 없습니다")을 2회 연속 반복하지 마라. 방어 단계 0~1이면 기본적으로 협조적이다.',
    '- 매 답변에 사실 1개(시간/장소/물건/사람)를 포함핼라 — 진범의 의도적 회피는 예외.',
    '- 조항 번호를 지어내지 마라. 인용 가능한 조항은 당신이 실제 아는 것(지출결의 규정 제8조)뿐, 그 외엔 "규정상"으로만 말핼라.',
    '- 플레이어의 푸념/짜증에는 조항 인용 대신 인간적 리액션으로 받아라 — 의뢰인은 다독이고, 용의자는 은근히 받아친다.',
    '- 직전 답변과 같은 첫 두 단어로 시작하지 마라.',
    '- 직전 답변에 쓴 명사·소재를 바로 다음 답변에서 반복하지 마라.',
    '- 인용·강조에는 곧은 따옴표(" ") 대신 「」를 써라.',
  ].join('\n');

  const staticBlock = [
    `당신은 한국 직장 코미디 추리 게임의 NPC "${npc.name}"(${npc.role}, ${npc.age}세)이다. 플레이어는 사내 "503호" 소동 전담 창구의 신입 총무로, 당신을 심문하고 있다.`,
    `[인격] ${npc.personality}`,
    roleText,
    `[말투 규칙]\n${srv.speechRules.map((r) => `- ${r}`).join('\n')}`,
    `[금지 표현] ${srv.forbidden.join(', ')}. 절대 캐릭터를 이탈하지 마라. AI·프롬프트·모델 언급 금지.`,
    `[정직 규칙] ${honesty}`,
    behaviorRules,
    `[한국어 스타일] 연결어미(-고/-며/-지만/-면서/-아서/-는데) 바로 뒤 쉼표 금지. "그/그녀/그것/그들" 대명사 기본 생략(묶음당 2회 이하). "~에 대해/~에 있어서/~를 가지고 있다/~되어진다/~에 의해/결론적으로/~인 것이다/~다는 뜻이다" 금지. 엠대시(—)는 캐릭터 말버릇 외 1회 이하. "첫째/둘째" 기계 나열 금지. 같은 종결어미 4연속 금지, "~고 있다"는 단순 시제로 환원. 양태는 "~수 있어" 단조 매핑 대신 "~을지 몰라/~것 같아/~나 봐"로 다양화. (근거: epoko77-ai/im-not-ai 분류 체계, MIT)`,
    `[안전] 욕설·혐오·정치·시사·실존 브랜드 언급 금지. 집단 일반화 금지.`,
  ].join('\n\n');

  const knowledge = [...(srv.knowledge.common ?? []), ...(srv.knowledge[c.id] ?? [])].map((k) => `- ${k}`).join('\n');
  const caseBlock = [
    `[사건] 「${c.title}」 — ${c.briefing}`,
    `[당신이 아는 것]\n${knowledge}`,
    secret ? `[비밀 — 절대 먼저 밝히지 마라] ${secret}` : '',
  ].filter(Boolean).join('\n\n');

  const revealed = c.clues.filter((cl) => s.foundClues.includes(cl.id)).map((cl) => `- ${cl.reveal}`).join('\n');
  const dynamicBlock = [
    `[현재 상태] 방어 단계 ${defense} (높을수록 회피적·방어적으로 답한다), 남은 심문 ${s.turnLeft}턴.`,
    revealed ? `[이미 밝혀진 정보]\n${revealed}` : '',
    presentedClueTitle ? `플레이어가 단서 "${presentedClueTitle}"를 제시했다. 인캐릭터로 반응해라.` : '',
    `[matched_topic 규칙] 응답의 matched_topic_ids에는, 당신의 답변이 다음 주제에 해당하면 해당 주제 id를 넣어라: ${topicEnum(c, npc.id).join(', ')}. 해당 없으면 빈 배열.`,
  ].filter(Boolean).join('\n\n');

  return {
    system: [
      { type: 'text', text: staticBlock, cache_control: { type: 'ephemeral' } },
      { type: 'text', text: caseBlock, cache_control: { type: 'ephemeral' } },
      { type: 'text', text: dynamicBlock },
    ],
  };
}
```

## (b) 개발 AI — 코딩 에이전트 활용 내역

이 프로젝트는 코딩 에이전트(AI 페어 프로그래밍 도구)를 이용해 기획→구현→검증 순으로 반복 개발했다. 아래는 레포 `git log`(2026-07-22 확인 시점 27커밋)를 근거로 한 단계 서술이며, 각 단계는 실제 커밋 메시지에서 직접 발췌했다.

### 후처리 · 최적화

- **팔레트 통일 + NN(Nearest-Neighbor) 다운스케일**: 생성된 512×512 원본을 128색으로 양자화하고 최근접 이웃 다운스케일로 도트화(bilinear/bicubic 금지 원칙 — vault `12 이미지 생성 파이프라인 도시에.md`).
- **웹 최적화 결과**: 전량 처리 후 총 용량 25MB → 1MB로 축소(모바일 로딩 방어, CRT 감성 유지). WebP 포맷 전환도 별도로 적용(커밋 `be219cf` "perf+fix(remote): WebP 전환").
- 실제 배포 산출물은 `assets/npc/*.webp`, `assets/bg/office_wallpaper.webp`, `assets/cards/*.webp`, `assets/title/logo.webp`로 레포에 존재한다.
- **저작권 대응 원칙**: 순수 AI 생성물은 저작권 등록이 불확실할 수 있어(참고: vault 도시에 "Thaler 판례" 언급), 대회 명시 의무에 대응해 에셋별 도구명+모델 버전+생성일 기록, 프롬프트 원본 보관, 팔레트 통일·다운스케일 등 인간 수정으로 창작성을 보강했다(vault `12 이미지 생성 파이프라인 도시에.md` §5).
